RAVOT — Dashboard 500-fout opgelost (JSON-groepering)

Oorzaak: het dashboard groepeerde de "geen resultaat"-zoekopdrachten op de
meta-kolom (type JSON). PostgreSQL kan JSON-waarden niet groeperen ("could not
identify an equality operator for type json"). Op SQLite (de tests) werkte het
wel, daarom kwam het pas op de live server naar boven.

Opgelost: het JSON-veld wordt nu naar tekst gecast vóór het groeperen —
werkt op zowel PostgreSQL als SQLite.

Dit was de reden dat /beheer een 500 gaf. Login + 2FA + QR waren allemaal OK;
de crash zat puur in het dashboard NA het inloggen.

Op de VPS na git pull:
  docker compose up -d --build
(geen migratie nodig)

Daarna: ravot.be/beheer/login -> inloggen -> QR scannen -> dashboard laadt.

Gewijzigd:
  app/routes/admin.py   (zero-result query: cast JSON naar tekst)
