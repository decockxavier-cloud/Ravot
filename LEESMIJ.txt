RAVOT — 2FA via QR-code (na login, maar 2FA blijft VERPLICHT)
==============================================================

Lost je 2FA-probleem op: je stelt 2FA nu in door een QR-code op het scherm te
scannen (geen secret meer overtypen in de terminal). 2FA blijft verplicht —
je wordt gedwongen het in te stellen bij de eerste login, je kunt het niet
overslaan. Alleen de MANIER van instellen is nu gebruiksvriendelijk.

HOE HET WERKT:
1. Ga naar /beheer, log in met e-mail + wachtwoord.
2. Heb je nog geen 2FA ingesteld? Dan verschijnt een QR-CODE op de pagina.
   Scan die met je authenticator-app (kies TIJDGEBASEERD / TOTP).
3. Typ de code ter bevestiging → klaar, je bent binnen.
4. Vanaf dan vraagt hij bij elke login gewoon je code (zoals het hoort).

BELANGRIJK — dit pakket bevat een nieuwe Python-package (segno, voor de QR).
Daarom MOET je herbouwen én migreren:

  cd /srv/ravot && git pull && docker compose up -d --build
  docker compose exec web flask migrate-db     # voegt admins.totp_confirmed toe (veilig)

Na de migratie staat je BESTAANDE admin automatisch op "nog niet bevestigd",
dus je doorloopt meteen de QR-flow — dat reset je kapotte 2FA vanzelf. 🎉

Je hoeft dus NIETS met de terminal te doen voor 2FA: gewoon inloggen op /beheer
en de QR scannen.

BONUS — nieuw commando als je ooit je telefoon kwijt bent:
  docker compose exec web flask reset-admin-2fa decock.xavier@gmail.com
  (genereert nieuwe secret; bij volgende login krijg je een verse QR)
